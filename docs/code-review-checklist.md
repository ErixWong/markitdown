# Code Review Checklist

> 基于任务队列实现审计中发现的典型错误提炼。

---

## 1. 数据完整性

- [ ] promote/remove 等重建队列操作是否保留了原始数据（不可用 None 占位）
- [ ] dequeue 返回值是否携带了下游所需的完整数据（不可只返回 ID 丢弃其余字段）
- [ ] 数据流是否闭环：enqueue 存了什么 → dequeue 取了什么 → 下游用了什么

## 2. 作用域与导入

- [ ] 模块级代码是否引用了仅在函数内部 `import` 的模块
- [ ] 条件分支中使用的 import 是否在所有分支可达的顶层
- [ ] 是否存在未使用的 import

## 3. 类型一致性

- [ ] 实际传参类型是否与函数签名的类型注解一致
- [ ] 是否存在隐式类型重载（如 `bytes | str` 通过 isinstance 区分但签名只写了 bytes）
- [ ] 静态类型检查（mypy/pyright）是否能通过

## 4. 变量覆盖

- [ ] 同一函数内是否对变量名复用，导致前一次赋值被覆盖
- [ ] `with self._lock:` 内部是否正确引用了外部变量

## 5. 索引与状态同步

- [ ] 索引赋值时机是否与数据结构变更同步（先 put 再记录 vs 先记录再 put）
- [ ] 异常路径（超时、拒绝）是否清理了所有已写入的索引/位置记录
- [ ] 索引重建是否使用增量更新而非全量遍历

## 6. 环境变量与配置

- [ ] 新增的环境变量是否在 `get_task_processor()` 等工厂函数中读取
- [ ] `.env.example` 是否同步更新
- [ ] 默认值是否与设计文档一致

## 7. API 层与业务层协作

- [ ] 错误状态是否正确传播到 HTTP 层（如队列满 → 503 而非静默 FAILED）
- [ ] 响应字段名是否与 API 文档/设计文档一致
- [ ] 异步操作的结果是否可被 API 层感知

## 8. 安全性

- [ ] Token 比较是否使用 `secrets.compare_digest()` 而非 `==`
- [ ] CORS 配置是否在生产环境收紧
- [ ] 管理员端点是否有独立的认证和速率限制

## 9. 测试覆盖

- [ ] 数据完整性路径是否有测试（promote 后数据是否可正常消费）
- [ ] 参数类型边界是否有测试（str 传给 bytes 参数等）
- [ ] 管理员 API 是否有认证/鉴权测试
- [ ] 并发竞态场景是否覆盖
